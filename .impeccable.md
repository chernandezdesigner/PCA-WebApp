## Design Context

### Users
Commercial property assessors and inspectors at an engineering/consulting firm. They use this tool at their desks after returning from field inspections, transcribing observations into structured report sections. They need to move efficiently through 37 form steps, entering technical data with minimal friction. The tool is internal-only — no public-facing users.

### Brand Personality
Reliable, efficient, professional. The interface should feel like a trusted work instrument — not flashy, not decorative, just clean and fast. Think: well-organized clipboard, not a marketing site.

### Aesthetic Direction
- **Visual tone:** Industrial-utilitarian with clean lines. Data density is a feature, not a bug.
- **Theme:** Both light and dark mode. Light mode is primary (office daytime use), dark mode for extended sessions.
- **Colors:** Blue-600 as primary action color. Semantic status colors (emerald=final, amber=review, purple=exported, slate=draft). Zinc neutrals for dark mode, slate neutrals for light mode.
- **Typography:** System font stack (Calibri loads via Lottie animation on splash, system-ui fallback). No display fonts needed — this is a data entry tool.
- **Anti-references:** Marketing dashboards with gradient cards, glassmorphism, decorative sparklines. This should NOT look like a SaaS landing page.

### Design Principles
1. **Density over decoration** — Every pixel should serve data entry or navigation. Remove visual weight that doesn't help the user complete their task.
2. **Consistent rhythm** — Use a strict spacing scale (4/8/12/16/24/32/48) and consistent border-radius across all components.
3. **Theme parity** — Every element must work in both light and dark mode. No hardcoded colors.
4. **Quiet interactions** — Transitions should be fast (150-200ms) and subtle. No bouncing, pulsing, or floating animations in a data-heavy form.
5. **Clear hierarchy** — Section headers > block labels > field labels > helper text. Use size and weight, never color tricks.
